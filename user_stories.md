User Stories for Smart Clinic Management System

1. Admin User Stories (System Management)

The Admin is responsible for the overall configuration, user provisioning, and system integrity.

As an Admin, I want to add new doctor accounts so that all practitioners are registered and available in the system for scheduling.

Acceptance Criteria (Example): The system validates the doctor's email is unique. A default, temporary password is generated.

As an Admin, I want to view and filter the list of all doctors by specialty so that I can quickly manage staffing levels and credentials.

As an Admin, I want to search for any patient's account so that I can resolve system access issues or verify user data efficiently.

As an Admin, I want to deactivate a doctor's account so that I can quickly secure the system when a staff member leaves.

2. Patient User Stories (Booking and Personal Management)

The Patient is focused on finding and managing their appointments.

As a Patient, I want to search for available appointments by date and medical specialty so that I can quickly find the right doctor when I need care.

As a Patient, I want to view my list of upcoming appointments so that I can easily prepare for my visits and manage my schedule.

As a Patient, I want to cancel an upcoming appointment so that I can free up the time slot for other patients.

As a Patient, I want to view my historical prescriptions so that I can access medication details without contacting the clinic.

3. Doctor User Stories (Scheduling and Patient Care)

The Doctor manages their own availability and patient medical history.

As a Doctor, I want to define and modify my weekly available time slots so that appointments can only be booked during my working hours.

As a Doctor, I want to view a list of all my scheduled appointments for a specific day so that I can manage my consultation queue efficiently.

As a Doctor, I want to view a patient's full record, including past appointments and prescriptions, so that I can make an informed diagnosis and prescribe accurate medication.

As a Doctor, I want to generate a new prescription document (MongoDB) so that the patient receives clear, digital instructions for their treatment.
